Les Commandes de Base de LINUX

La maîtrise des commandes de base de Linux est essentielle pour les bioinformaticiens. Voici un guide simplifié et illustré pour vous aider à comprendre et utiliser efficacement ces commandes.

1. Gestion des répertoires et des fichiers

Commande Description Exemple
pwd Affiche le chemin absolu du répertoire courant. pwd → /home/user/Documents
ls Liste les fichiers et dossiers du répertoire actuel. ls → file1, file2, etc.
cd Change de répertoire. cd Documents
mkdir -p Crée un répertoire, y compris les sous-dossiers. mkdir -p bioinfo/data
touch Crée un fichier vide ou met à jour la date de modification. touch fichier.txt
cp Copie des fichiers. cp fichier.txt /home/user/Desktop
cp -r Copie un répertoire et son contenu. cp -r dossier1 dossier2
mv Déplace ou renomme des fichiers. mv fichier.txt nouveau_nom.txt
rm Supprime des fichiers (irréversible). rm fichier.txt
rm -R Supprime un répertoire et tout son contenu. rm -R dossier

2. Commandes d'édition

Commande Description Exemple
cat Affiche le contenu d’un fichier. cat fichier.txt
more Affiche le contenu d’un fichier page par page (sans retour arrière). more fichier.txt
less Comme more, mais permet un défilement arrière. less fichier.txt
head Affiche les premières lignes d’un fichier. head -n 5 fichier.txt → Affiche 5 premières lignes.
tail Affiche les dernières lignes d’un fichier. tail -n 5 fichier.txt → Affiche 5 dernières lignes.
nano Éditeur de texte simple en terminal. nano fichier.txt
gedit Éditeur graphique pour les fichiers texte. gedit fichier.txt
grep Recherche une chaîne dans un fichier. grep "motif" fichier.txt
echo Répète une chaîne de caractères dans le terminal. echo "Bonjour!"

3. Commandes diverses

Commande Description Exemple
cal Affiche le calendrier du mois courant. cal
date Affiche la date et l’heure actuelles. date
wc Compte le nombre de lignes, mots et caractères dans un fichier. wc fichier.txt
grep Recherche une chaîne dans des fichiers ou répertoires. grep "motif" fichier.txt
spell Vérifie l'orthographe dans un fichier. cat texte.txt | spell > erreurs.txt
read Lit une entrée utilisateur (dans les scripts). read nom → Attend que l’utilisateur tape un texte.

Commandes de redirection Bash pour débutants

Les redirections en Bash permettent de gérer l'entrée et la sortie des commandes. Voici quelques commandes de base :

1. Redirection de la sortie standard

Pour envoyer la sortie d'une commande vers un fichier en écrasant son contenu :

commande > fichier.txt

Exemple :

echo "Bonjour" > salut.txt

Pour ajouter (append) la sortie d'une commande à la fin d'un fichier sans écraser son contenu :

commande >> fichier.txt

Exemple :

echo "Au revoir" >> salut.txt

2. Redirection de l'entrée standard

Pour utiliser le contenu d'un fichier comme entrée d'une commande :

commande < fichier.txt

Exemple :

sort < unsorted.txt

3. Redirection de la sortie d'erreur

Pour rediriger les messages d'erreur d'une commande vers un fichier :

commande 2> erreurs.txt

Exemple :

ls dossier_inexistant 2> erreurs.txt

4. Redirection de la sortie standard et d'erreur

Pour rediriger à la fois la sortie standard (stdout) et la sortie d'erreur (stderr) vers un même fichier :

commande &> sortie.txt

Ou en utilisant la syntaxe équivalente :

commande > sortie.txt 2>&1

5. Here Document (redirection multi-ligne)

La redirection "Here Document" permet de fournir plusieurs lignes de texte en entrée à une commande :

commande << EOF
ligne 1
ligne 2
...
EOF

Exemple :

cat << EOF
Bonjour,
Ceci est un exemple.
EOF

6. Here String

Utilisez "Here String" pour envoyer une chaîne de caractères comme entrée à une commande :

commande <<< "texte"

Exemple :

grep "motif" <<< "Ceci est un texte contenant le motif."

Récapitulatif

  • > : Redirige la sortie standard vers un fichier (écrase le fichier).
  • >> : Ajoute la sortie standard à un fichier (sans écraser).
  • < : Utilise le contenu d'un fichier comme entrée.
  • 2> : Redirige la sortie d'erreur vers un fichier.
  • &> ou > fichier 2>&1 : Redirige à la fois la sortie standard et la sortie d'erreur.
  • Here Document : Fournit plusieurs lignes comme entrée.
  • Here String : Fournit une chaîne de caractères comme entrée.
